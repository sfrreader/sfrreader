Extended licence obtaining

Following the agreement with SFR System manufacturer, to embarrace unauthorised base unit programming and competition vandalism, one willing to obtain his first short-term extended license should provide proof of SFR base unit ownership.

Picture with base unit and "SFR Reader" text sent to the srgttv@gmail.com would be a valid proof.

Reading card data

When launched, SFR Reader would propose to turn NFC on. Than you just should apply card to the smartphone to read data.

Applying the same card for the second time would result in message "The data are read already".

Setting card's owner

SFR Reader allows to set team/participant name for each card it reads.

One can set it manually after each reading operation, or in semi- and even fully automatic way. To speed the process up one can upload participant list from a text file (UTF-8 encoded) or Event Centre database, to choose correct name from the list. If the participant in the list has a card number, than SFR reader would automatically use that name for the card.

You can use Excel or similar program (Open / Libre Office) to prepare the list and save it as CSV-file.

Sample: 
// Line starting with 2 slashes is a comment.
// The file format is trivial: 
// One line — one team

// These teams should be  
// chosen manually
The Furious
Just walking
Quantum snails

// For the following teams
// SFR Reader would use
// card #32 for Swamp kids, 
// and card #34 for John Doe.
// Text befor semicolon is a card number, 
// text after it is a team name.

31;Chasing daylight
32;Swamp kids
33;Humpty Dumpty
34;John Doe
35;Heather Jones
36;Salmon - 1
37;Cheetah - 7

Participants list upload

To import participants list upload file to the smartphone (via e-mail, messenger, USB or Bluetooth), choose "Load teams..." menu item, and find uploaded file.

SFR Reader would display number of uploaded teams and proceed to the teams list, or show an error message for ill-formed files.

Base unit programming

Base units are programmed in 3 steps using a master-card.
  1. Choose what to program.
  2. Apply card to the phone.
  3. Apply card to the base unit.
Average programming time: less than 1 second.

Known issues

Some smartphones are known to have very slow NFC data transfer (1-2 seconds per 1 punch, instead of regular 30-100 msec).

The problem models are:

  • Alcatel POP 4S (5095Y)
  • Sony Experia Compact Z3 (D5803)
